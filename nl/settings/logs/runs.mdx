_Wanneer een automatisering wordt getriggerd door een event vanuit je gekoppelde app, bijv. Carerix, Salesforce, enz., wordt een Run aangemaakt. Deze Run vertegenwoordigt het volledige proces dat volgt vanaf de initiële trigger tot elke stap van je automatisering. In dit artikel behandelen we wat runs zijn, hoe je ze vindt en leest, en hoe ze je kunnen helpen je automatiseringen te troubleshooten._

## Wat is een run

Een run is het resultaat van een getriggerde automatisering. Deze toont de exacte sequentie van acties die plaatsvonden nadat een event werd ontvangen, bijvoorbeeld een filter dat wordt gecontroleerd, een vertraging die is voltooid, of een e-mail die is verzonden/gesprek dat is beëindigd. Elke Run bevat een volledige timeline van de genomen stappen, een **Overview**, **Fields**, en **Related**-tabbladen.

Zodra een run succesvol is voltooid, wordt deze weergegeven op de pagina **Task History** door een nieuwe taak aan de lijst toe te voegen.

## Hoe je details van runs bekijkt

Om een individuele run te verkennen, klik je op de knop [Details] naast de run voor een kort overzicht of open je het dropdown-menu en klik je op [View Run].

![Hoe je individuele runs bekijkt](/images/intercom/49924583-Screenshot-2025-05-05-at-10_54_51.png)

Dit opent de detailweergave van de run, die het volgende bevat:

- **Overview** – een samengevat overzicht van de data die van je app is ontvangen.

![Run Overview-tabblad](/images/intercom/49924603-Screenshot-2025-05-05-at-10_55_16.png)

- **Fields** – velden met de data in het event dat de automatisering heeft getriggerd en mogelijk in de run is gebruikt.

![Run Fields-tabblad](/images/intercom/49924650-Screenshot-2025-05-05-at-10_55_27.png)

- **Related** – de automatisering die deze run volgde en het event dat het triggerde.

![Run Related-tabblad](/images/intercom/49924666-Screenshot-2025-05-05-at-10_55_33.png)

Deze gedetailleerde weergave stelt je ook in staat om precies te zien hoe je automatisering is verlopen en of elke stap is geslaagd.

## Troubleshooting van runs

Als er iets niet is gebeurd zoals verwacht:

- Open de detailweergave van de run om logs te bekijken en te zien of een stap is mislukt of overgeslagen.
- Open het tabblad **Fields** om te bevestigen of de data die in de run is gebruikt aan je voorwaarden voldeed. Je kunt ook de run vergelijken met het oorspronkelijke event (te vinden via het tabblad **Related**) om te begrijpen wat het triggerde en te bevestigen dat de juiste data is gebruikt.

## Toch nog hulp nodig?

Als je niet zeker weet waarom een run is geannuleerd/voltooid of hulp wilt bij het interpreteren van de stappen van de run, neem dan gerust contact op met ons team via [support@hiredata.com](mailto:support@hiredata.com). Voeg als je dat doet toe:

- Een link of het ID van de run;
- Alle relevante data of context die ons kunnen helpen verder te onderzoeken.
