_Diese Anleitung führt Sie in alle Grundlagen des [Task History]-Tabs ein, einschließlich der Verwendung der Filter, um abgeschlossene und abgebrochene Aufgaben zu sehen, wo Sie Event-Details finden und wie Sie diese erneut abspielen. Scrollen Sie ans Seitenende, wenn Sie das Videoformat bevorzugen._

Um Aufgaben im [Task History]-Tab zu sehen, müssen [Automatisierungen](/de/automations/automations) durch HireData laufen. Hier sehen Sie alle Aufgaben, die sie ausgeführt haben.

![Task History tab](/images/intercom/0ef47336b95b.png)

## So filtern Sie Aufgaben

Eine der Funktionen dieses Tabs sind die [Filters]. Sie ermöglichen es, Aufgaben nach abgeschlossen oder abgebrochen zu filtern.

_Ansicht abgeschlossener Aufgaben:_

![Completed tasks view](/images/intercom/3ca8d7257ea7.png)

Um nur die abgebrochenen Aufgaben zu sehen, aktivieren Sie den Cancelled-Filter, indem Sie den Schieberegler nach links bewegen, sodass er grau wird.

_Ansicht abgebrochener Aufgaben:_

![Cancelled tasks view](/images/intercom/afe4efe9c027.png)

## Events verstehen

Wenn Sie prüfen möchten, warum eine Aufgabe abgebrochen wurde, können Sie den Event-Verlauf ansehen. Klicken Sie neben der zu untersuchenden Aufgabe auf [View Event], und Sie sehen die Details.

![Cancelled task with View Event and View Process buttons](/images/intercom/932cef706d97.png)

_Beispiel-Event-Details:_

![Event details](/images/intercom/ca09fd4105a6.png)

Wenn Sie auf [Processes] klicken, sehen Sie alle Prozesse, die durch dieses Event erzeugt wurden. In diesem Event können wir zum Beispiel sehen, dass das Event die angezeigte Automatisierung durchlief, am Ende jedoch aufgrund ungültiger Daten abgebrochen wurde.

_Beispielhafte Event-Prozesse:_

![Event processes](/images/intercom/a7d9fc71346f.png)

Diese Übersicht gibt Ihnen eine Vorstellung davon, was passiert ist und ob Fehler vorlagen, die die Aufgabe am Abschluss gehindert haben. So können Sie sie beheben und die Aufgabe abschließen.

## So spielen Sie ein Event erneut ab

Wenn Sie eine Aufgabe bearbeiten und sehen möchten, ob sie abgeschlossen wird, können Sie das Event erneut abspielen, indem Sie im Dropdown-Menü auf die Schaltfläche [Replay] klicken.

Wenn Sie zum Beispiel im Processes-Tab des Events sehen, dass die in der Automatisierung eingerichtete Verzögerung nicht funktioniert, können Sie sie entfernen und das Event dann erneut abspielen, um zu sehen, was passiert.

![Event Replay button](/images/intercom/686b1bc6c45d.png)

## Profi-Tipp

Die Zahlen neben dem [Task History]-Tab werden aktualisiert, wenn Sie zwischen abgeschlossenen und abgebrochenen Aufgaben wechseln, und zeigen die Gesamtzahl der Aufgaben an.

![Total number of completed tasks](/images/intercom/0bbd33e5c373.jpg)
